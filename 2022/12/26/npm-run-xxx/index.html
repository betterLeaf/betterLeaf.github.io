npm run xxx的时候发生了什么

Created2022-12-26|Updated2022-12-27
|Post View:

结论

  1. 运行 npm run xxx的时候， npm 会先在当前目录的node_modules/.bin查找要执行的程序，如果找到则运行
  2. 如果没有找到，则从全局node_modules/.bin中查找, npm i -g xxx就是安装到全局目录
  3. 如果还是没找到，那么就从path环境变量中查找有没有其他同名的可执行程序

实现原理

npm 实现的原理非常简单。 每当执行npm run, 就会新建一个Shell,在这个Shell里面执行指定的脚本命令.
比较特殊的是，npm run 新建的这个Shell，会将当前目录的node_modeules/.bin子目录加入PATH变量，执行结束后，再将PATH恢复原样
所有脚本都是可以直接使用脚本名调用，而不必加上路径

常见问题

为什么不直接执行 vue-cli-service serve而要执行npm run serve呢

  • 因为直接执行vue-cli-service serve会报错，因为操作系统中不存在这一条指令，当执行npm run xxx的时候会多一个查找的操作，去查找xxx对应的可执行文件
  • npm i 的时候, npm 已配好这种软连接，这种软连接相当于一种映射，执行npm run xxx的时候，就会到node_modules/bin中查找对应的映射文件，然后再找到对应的js赖执行

在node_modules/bin中有三个相同名称的脚本文件
